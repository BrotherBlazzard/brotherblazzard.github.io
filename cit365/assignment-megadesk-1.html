Instructions

  1. Review the design documents and conversations from this week's Developer Forum and discussion board.
  2. Create a new desktop project in Visual Studio.
  3. Name the project MegaDesk or something similar. We will be building upon this version for next week and the week after. You will need to create and maintain this project as a separate repo.
  4. Build your application framework with default support files and an initial form named Form1.cs.
  5. Rename Form1.cs to MainMenu.cs and build a main menu interface as shown in the MegaDesk Project Description.
  6. Add the following objects:
    1. AddQuote.cs (Windows Form): This form provides the user input interface.
    2. DisplayQuote.cs (Windows Form): This form view displays the current quote information after a successful quote add.
    3. ViewAllQuotes.cs (Windows Form): This form will display all the quotes by reading from the a text file where quotes are stored.
    4. SearchQuotes.cs (Windows Form): This user interface form allows the user to pick a material type from a combo box and displays all quotes with that material.
    5. Desk.cs (Class): This class will define the attributes of the desk object with width, depth, number of drawers, and the surface material may contain validation of those attribute values.
    6. DeskQuote.cs (Class): This class will define the attributes of a quote including Desk, rush days, customer name, and quote date. This class will also hold the logic in determining the line item cost.
  7. Create a 3 min (or less) video demonstrating the navigation and validation of the inputs. You do NOT need to show code unless there is something specific you want to demonstrate. Narrate your video and demonstrate that your application meets the requirements.
  8. Sync your finished application to a new properly labeled GitHub repository. Make sure it is complete from the project root and includes the .sln solution file.

Application Requirements

You are not completing the entire MegaDesk application this week. Rather, you are just getting started by planning and designing the user interface (forms) and setting up the classes with the appropriate logic. In this version it is OK to use hard coded 'magic numbers' such as base price, minimum width, rush order days, and values for combo boxes, etc. The next version will add additional development specifications and requirements to meet the remaining functional specifications from the design.

  1. Your application must have the form navigation completed, allowing the user to navigate from the main menu to each form and back again, and the user should be able to exit the application.
  2. Validate the width input with a Validating event that checks for an integer input and that checks for valid minimum and maximum values. The application should provide visual clues for the user to identify where the error is located.
  3. Validate the height input using a KeyPress or related event that checks for valid integer input using a compound statement and the char data type with IsControl and IsDigit methods.
  4. Be sure to consider proximity, alignment, contrast, and padding in your input and output interface designs. PARC Principles review.
