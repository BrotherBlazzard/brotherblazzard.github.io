Lesson 06 - ASP.NET Web Pages and Data

Overview

This tutorial shows you how to create a database in Visual Studio and how to display database data in a page when you use ASP.NET Web Pages (Razor). It assumes you have completed the series through Introduction to ASP.NET Web Pages Programming. Now that you have an understanding of the basics of ASP.NET Web Pages and the Razor syntax, you are ready to build a small application that will be used in the development exercise in this lesson and subsequent lessons. The application deals with a movie database where the user has the ability to view, add, edit, and delete information about their favorite movies.

Brief Database Review

Typically a database contains one or more tables of information, data that has been organized into rows and columns with headers. These tables represent entities within the application that we are developing. When there is more than one table in a database, those tables are normally related and follow rules to keep the data manageable. This tutorial contains a simple table of information that tracks movies, which can be the title of our table in the database. Initially our movies table will contain columns such as title, genre, and year (meaning year released) for each movie. Each movie record is a row in the database table. The columns or fields are the attributes that describe the movie records.

As part of the normalization of information in a database, each table or entity contains a primary key or unique value that is used to identify a single record or row in the table. Primary keys can be made up of one or more of the existing columns or they can be a developed column if none of the columns suffice as an appripriate unique attribute.

In dynamic web applications and sites the displayed content is driven form a database. The most common operations in a dynamic web applications are viewing, adding, editing, and deleting information.
