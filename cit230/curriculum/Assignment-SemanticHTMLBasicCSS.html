Overview

The assignment objective is to apply the concepts from the lesson to create your home page for the course. This page will be named index.html which is a standard, default name for the root file of a directory on popular web servers. The page will be an assignment portal which will be published (uploaded) to your GitHub Pages account at your username.github.io and you will continue to update this page throughout the semester with assignment links.

Instructions

❏ Launch VS Code and ...

❏ Connect to GitHub Pages using VS Code or use GitHub Desktop - EXAMPLE NEEDED

❏ Create a new file and name it index.html.

❏ Start your page with a proper HTML 5 document type declaration (which is not a HTML tag) to let the browsing agent know what version of HTML is being used in the page.

❏ Write the opening html tag with the lang attribute. This attribute declares the written language of document contents. Please note the case and spacing.

This attribute of the html element declares the language of the html document and provides users, user agents and readers information about the written language of the document. The first part of the lang attribute value is the language, and the second part following the dash is the country, if needed. Addition Information: w3.org (TL;DR)

'TL;DR' means Too Long; don't read which does not mean don't read this but rather a warning that it is a long document with probably more information and opinion that you really needed to know and is considered optional in the course.

❏ Write the <head></head> and the <body></body> container elements. The head contains information (metadata) about the page. The body element contains the displayable content of the page.

❏ Set the charset meta tag to enable universal character encoding. Note that this element is self closing meaning there is no closing tag.

This standard charset covers almost all the characters that you would find used in the world. It is written as the first meta item in the head.

❏ Next Set the viewport meta tag to control the view-able area. This tag follows the charset meta tag.

<meta name="viewport" content="width=device-width, initial-scale=1">

The viewport is the visible area of a page. Not all devices and browsers render pages the same. This meta tag provides control over the width and scale of the page in the browser.
"Using the viewport meta tag to control layout on mobile browsers": MDN (TL;dr)

❏ Write the document title with this content substituting your name. Note, that this tag has a corresponding closing tag.

<title>CIT 230 Assignment Portal | Chris B. Student | BYU-Idaho</title>

Browse and consider this article going forward when titling pages: "What is a title tag?"

| or the pipe character is a favorable separator, if a separator is needed, in title tag content. This character is found on standard keyboards and may appear as a dashed, vertical line above the enter or return key.

❏ Describe the page content by adding the meta description tag.

<meta name="description" content="...">

The meta description tag provides information about the page's content. Use keywords in the construction of your description which is placed in the content attribute value (" ... "). Example, "Assignment portal page for Chris Student in CIT 230: Web Frontend Development at Brigham Young University-Idaho."

❏ Add semantic structural elements like <header> and <footer>

❏ Include the following elements and content to the appropriate block elements.

  • h1 heading containing your full name within the header.
  • h2 heading within the main element as the first item with this content "CIT 230 - Assignment Portal."
  • An unordered list directly after the h2 heading with at least one placeholder list item that contains a link placeholder to future assignments pages.
<li>Lesson ##: <a href="#">Assignment Label</a></li>
  • A profile image into the header before the h1 tag. Note that you will have to place your image into the proper directory, match the name you select, and have a matching reference in your img src attribute. If your image does not show up when viewing your page through a browser, typically one of this three items is incorrect.
  • Add the following in one line within a paragraph tag in the footer
    • an HTML entity copyright symbol followed by the year,
    • your full name,
    • your state or country, and
    • a link to BYU-Idaho Online Learning.

❏ Add styling to the page • use an external CSS file

<link rel="stylesheet" href="styles/main.css">

Add this statement to your head after the description meta tag. You will need to create a new file in the styles folder named main.css. This is an external style sheet, and it will contain some basic CSS declarations upon the elements we have in our html document.

✂ Use this CSS template to develop your main.css file. Replace the comments in the CSS rules with your own values. Make sure to keep the markup clean and uniform.

When you click the CSS template link, the text file will open in a new tab. Copy the text and paste it into your main.css file in your editor.

❏ Commit and push your assignment portal home page to your remote GitHub Pages repository.

Push up your updated site with all the requisite directories (folders) and files including the index.html file and the main.css page making them available for assessment.
